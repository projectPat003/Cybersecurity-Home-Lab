# Threat Intelligence Reports

Structured threat intelligence reports covering real-world CVEs, APT campaigns, malware campaigns, and critical infrastructure threats. Each report follows a standardized five-section format aligned with professional threat intelligence workflows. This section also documents the automated daily threat intelligence pipeline running 24/7 against live attack data captured by the honeypot.

---

## Report Format

All manual reports follow this five-section structure:

| Section | Description |
|---|---|
| Overview | Vulnerability or threat summary, CVSS score, affected versions, and threat actor context |
| How the Attack Works | Technical breakdown of the exploit mechanism or attack chain |
| Impact | CIA triad mapping and real-world consequences |
| Detection | IOCs, SIEM detection opportunities, Wazuh rule opportunities, and AbuseIPDB enrichment |
| Recommended Actions | Patches, mitigations, and hardening steps |

---

## Automated Daily Threat Intelligence Report

In addition to manual reports, an automated threat intelligence pipeline runs every morning at 9 AM EST and posts a structured report to Discord. This report pulls live data from the Wazuh OpenSearch indexer covering the previous 24 hours of attack activity against the honeypot and monitored endpoints.

**Report sections:**

- AI executive summary written by Claude Haiku analyzing the day's threat landscape
- Alert volume breakdown by severity: Critical, High, Medium, Low
- Alerts broken down by agent
- Busiest attack hour of the day
- Top 5 attacking IPs enriched with AbuseIPDB data including abuse score, country, ISP, and total prior reports
- Top 15 usernames attempted in SSH brute force attacks
- MITRE ATT&CK techniques observed with hit counts
- File integrity event summary
- Top triggered Wazuh rule IDs
- Active agent status

The report is generated by `scripts/daily-threat-report.py` which queries the Wazuh API, calls the AbuseIPDB API for IP enrichment, and uses the Anthropic Claude Haiku model to generate the AI executive summary before posting everything to Discord in chunked messages.

## Tools and Frameworks Referenced

- **MITRE ATT&CK** — TTP mapping for all reports and automated daily alerts
- **CVSS v3.1** — Severity scoring for manual CVE reports
- **Wazuh** — Detection rule development and live alert sourcing for the daily report
- **AbuseIPDB** — IP threat intelligence enrichment for SSH brute force alerts and daily report top IPs
- **Anthropic Claude API** — AI-generated executive summaries in the daily threat report
- **Discord** — Delivery platform for automated daily reports and real-time alerts
- **CISA Advisories** — Threat intelligence sourcing for manual reports
- **Shodan** — Passive reconnaissance and honeypot exposure verification
